Week 12 — False Positive Analysis & Threshold Validation

Hours Logged: 13

Objective

Improve reliability of static SQL warnings by introducing configurable correlation thresholds and implementing a diagnostic framework to identify and explain false positives using execution plan evidence.

Activities

Implemented a configurable threshold strategy controlling runtime confirmation criteria, including minimum scan count, estimated row thresholds, and operator-based confirmation policies. Developed a false-positive diagnostic module that analyzes static warnings lacking runtime support and produces structured explanatory reports. Integrated threshold configuration into the correlation engine to enable systematic tuning and reproducible validation behavior. Executed runtime validation experiments on the execution plan workload and generated a diagnostic validation log summarizing confirmation outcomes.

Outcome

QueryLens now includes an explainable runtime validation layer with tunable confirmation criteria and automated false-positive diagnostics. The system supports empirical threshold calibration and provides transparent evidence describing when and why static warnings are or are not confirmed by execution plan behavior.
