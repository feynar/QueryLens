Week 16 — Expanded Runtime Validation and Workload Evaluation

Hours Logged: 24

Objective

Evaluate the effectiveness of QueryLens static SQL anti-pattern detection by validating whether detected warnings correspond to observable runtime behaviors in SQL Server execution plans.

Activities

Expanded the SQL workload to 30 queries covering a broader range of query structures including joins, aggregations, filtering conditions, sorting operations, subqueries, and window functions. Generated SQL Server execution plans for each query using SHOWPLAN_XML and collected execution plan artifacts for runtime analysis.

Enhanced the execution plan parsing prototype to extract runtime operator information including clustered index scans, index seeks, hash joins, and explicit sort operations. Implemented an expanded runtime validation module that compares static warnings generated by QueryLens with execution plan evidence.

Executed the validation pipeline across all queries in the expanded workload and generated structured evaluation artifacts summarizing warning counts and runtime confirmations. Integrated the validation results into the experimental evaluation dataset to support quantitative analysis of the relationship between static anti-pattern detection and observed execution plan behavior.

Outcome

The expanded evaluation produced 17 static SQL performance warnings across the workload. Of these warnings, 13 were confirmed through corresponding execution plan behaviors, resulting in a runtime confirmation rate of approximately 76.5%.

These results demonstrate that QueryLens static analysis rules frequently correspond to measurable execution plan characteristics, providing empirical support for the system’s static-to-runtime correlation approach.
